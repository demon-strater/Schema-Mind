# SchemaMind — 상세 기획서

**서비스명:** SchemaMind (스키마마인드)
**부제:** Digital Brain · 계층적 지식 구조화 서비스
**버전:** 1.0
**최종 수정일:** 2026-03-11

---

## 1. 서비스 개요

### 1.1 비전

SchemaMind는 인간의 사고 과정을 모방한 **"디지털 두뇌"** 서비스이다. 사용자가 텍스트(논문, 기사, 학습 노트 등)를 붙여넣으면, AI가 자동으로 해당 텍스트를 **한국십진분류법(KDC)** 기반 9개 대분류 카테고리 중 적합한 분야에 배치하고, **DIKW 피라미드**(Data → Information → Knowledge → Wisdom) 위계에 따라 내용을 분해·구조화하여 **방사형 마인드맵**으로 시각화한다.

### 1.2 핵심 가치

| 가치 | 설명 |
|------|------|
| **자동 분류** | 사용자가 분야를 지정할 필요 없이 AI가 9개 학문 카테고리 중 자동으로 적합한 영역을 판별 |
| **지식 계층화** | 원문 텍스트를 지혜(Wisdom), 지식(Knowledge), 정보(Information), 데이터(Data) 4단계로 분해 |
| **시각적 사고** | SVG 기반 전체화면 방사형 마인드맵으로 지식 간 관계를 직관적으로 탐색 |
| **연결 발견** | AI가 서로 다른 계층 간 숨겨진 연결 관계를 자동 식별하여 점선 링크로 표시 |

### 1.3 대상 사용자

- 학습 내용을 체계적으로 정리하고 싶은 **학생 및 연구자**
- 다양한 분야의 지식을 통합적으로 관리하고 싶은 **지식 노동자**
- 읽은 문서의 핵심을 빠르게 추출하고 싶은 **독서가**
- 아이디어와 정보를 시각적으로 구조화하고 싶은 **기획자 및 크리에이터**

## 2. 정보 구조 (Information Architecture)

### 2.1 노드 계층 구조

SchemaMind는 8단계의 계층 구조를 사용한다. 중앙의 "나(Cogito)"에서 시작하여 점점 구체적인 정보로 세분화된다.

```
Level 0: Cogito (나) — 최상위 루트, 사용자 자신
  └─ Level 1: 대주제 (Subject) — KDC 기반 9개 고정 카테고리
       └─ Level 2: Article (기사/논문) — AI가 생성한 원문 요약 노드
            └─ Level 3: 지혜 (Wisdom) — 핵심 원리, 철학적 통찰, 보편적 진리
                 └─ Level 4: 지식 (Knowledge) — 이론, 프레임워크, 구조화된 개념
                      └─ Level 5: 정보 (Information) — 맥락적 사실, 정의, 설명
                           └─ Level 6: 데이터 (Data) — 원시 사실, 수치, 인용, 날짜
                                └─ Level 7: 원문 (Raw) — 확장용 예비 계층
```

### 2.2 KDC 기반 9개 대분류 카테고리 (Level 1)

한국십진분류법(KDC)을 기반으로 인류 지식의 전 영역을 9개 카테고리로 분류한다. 이 카테고리는 시스템에 **고정**되어 있으며 사용자가 추가하거나 삭제할 수 없다.

| 번호 | 카테고리 | 영문명 | 설명 | 아이콘 |
|------|----------|--------|------|--------|
| 1 | 철학 | Philosophy | 존재, 인식, 가치, 논리 등 근본적 질문을 탐구하는 학문 | 🧠 |
| 2 | 종교 | Religion | 신앙, 영성, 종교적 사상과 의례에 관한 학문 | 🕊️ |
| 3 | 사회과학 | Social Science | 인간 사회의 구조, 제도, 행동을 연구하는 학문 | 🏛️ |
| 4 | 자연과학 | Natural Science | 자연 현상을 관찰, 실험, 체계적으로 연구하는 학문 | 🔬 |
| 5 | 기술과학 | Technology | 과학적 지식을 실용적으로 응용하는 학문과 기술 | ⚙️ |
| 6 | 예술 | Arts | 미적 표현과 창작 활동에 관한 학문 | 🎨 |
| 7 | 언어 | Language | 언어의 구조, 사용, 습득에 관한 학문 | 📝 |
| 8 | 문학 | Literature | 문학 작품의 창작, 비평, 연구에 관한 학문 | 📚 |
| 9 | 역사 | History | 과거의 사건, 인물, 문화를 기록하고 해석하는 학문 | 📜 |

### 2.3 DIKW 위계 체계

AI가 텍스트를 분석할 때 내용을 4개 계층(Tier)으로 분류한다. 각 Tier는 마인드맵에서 부모-자식 **체인(Chain)** 형태로 연결된다.

```
Article (L2): 원문 전체 요약
    │
    ├── 💡 지혜 (Wisdom, L3): 핵심 교훈, 철학적 통찰
    │       "양자역학의 핵심 교훈은 관측이 대상의 상태에 영향을 미친다는 것이다."
    │
    │   └── 📖 지식 (Knowledge, L4): 이론적 체계, 프레임워크
    │           "하이젠베르크의 불확정성 원리는 위치와 운동량을 동시에 측정할 수 없다."
    │
    │       └── ℹ️ 정보 (Information, L5): 맥락적 사실, 정의
    │               "양자역학은 미시세계의 물리현상을 설명하는 물리학의 한 분야이다."
    │
    │           └── 📊 데이터 (Data, L6): 수치, 날짜, 원시 사실
    │                   "플랑크 상수 h = 6.626 × 10⁻³⁴ J·s"
    │
    └── (같은 Tier의 복수 노드는 동일 부모 아래 형제로 배치)
```

**체인 구조 규칙:**
- 같은 Tier의 여러 항목은 동일 부모 아래에 **형제(sibling)** 로 배치
- 다음 Tier의 첫 번째 항목은 이전 Tier의 첫 번째 항목을 **부모**로 삼음
- 이를 통해 Wisdom → Knowledge → Information → Data 순서의 수직적 위계가 형성됨

### 2.4 노드 데이터 모델

각 노드는 다음 속성을 가진다:

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | serial (자동 증가) | 고유 식별자 |
| `parentId` | integer (nullable) | 부모 노드 ID (null이면 최상위) |
| `level` | integer (0-7) | 계층 깊이 |
| `title` | text | 노드 제목 |
| `description` | text (nullable) | 간략 설명 |
| `content` | text (nullable) | 상세 내용 (Article 노드에 원문 저장) |
| `color` | text (nullable) | 표시 색상 (hex) |
| `icon` | text (nullable) | 아이콘 이름 |
| `sortOrder` | integer | 형제 간 정렬 순서 |
| `createdAt` | timestamp | 생성 시각 |

### 2.5 연결(Connection) 데이터 모델

부모-자식 관계 외에, 노드 간 **횡적 연결(cross-link)** 을 별도로 저장한다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | serial | 고유 식별자 |
| `sourceId` | integer | 출발 노드 ID |
| `targetId` | integer | 도착 노드 ID |
| `description` | text (nullable) | 연결 사유 |
| `createdAt` | timestamp | 생성 시각 |

## 3. 핵심 기능 상세

### 3.1 AI 텍스트 분석 (자동 분류 엔진)

#### 3.1.1 기능 설명

사용자가 텍스트를 입력하면 GPT-4o 모델이 다음 작업을 수행한다:

1. **카테고리 판별:** 9개 KDC 카테고리 중 가장 적합한 1개를 선택
2. **요약 생성:** 원문을 대표하는 Article 제목과 한 줄 설명 생성
3. **DIKW 분해:** 텍스트의 모든 정보를 지혜/지식/정보/데이터 4단계로 분류
4. **관계 식별:** 서로 다른 항목 간 의미적 연결 관계를 자동 발견

#### 3.1.2 AI 프롬프트 설계

```
역할: "You are a knowledge architect."

입력: 사용자 텍스트 (최대 50,000자)

출력 (JSON):
{
  "category": "9개 중 정확한 카테고리명",
  "articleTitle": "간결한 제목",
  "articleDescription": "한 줄 요약",
  "items": [
    {
      "title": "항목 제목",
      "description": "간략 설명",
      "content": "상세 내용 또는 텍스트 발췌",
      "tier": "wisdom | knowledge | information | data"
    }
  ],
  "connections": [
    { "from": "항목 제목", "to": "항목 제목", "reason": "연결 사유" }
  ]
}

제약 조건:
- category는 반드시 9개 카테고리 중 정확한 이름
- 4-8개 항목, 4개 Tier에 골고루 분배
- 가능한 한 모든 Tier에 최소 1개 항목 포함
```

#### 3.1.3 분석 후 데이터 생성 플로우

```
1. AI 응답 JSON 파싱 및 Zod 스키마 검증
2. 카테고리 매칭: 정확 일치 → 부분 일치 → 첫 번째 카테고리 폴백
3. Article 노드(L2) 생성 (부모 = 매칭된 카테고리)
    - content 필드에 원문 전체 저장
4. Tier별 노드 체인 생성:
    - wisdom 항목들 → L3 노드 (부모 = Article)
    - knowledge 항목들 → L4 노드 (부모 = 첫 번째 wisdom 노드)
    - information 항목들 → L5 노드 (부모 = 첫 번째 knowledge 노드)
    - data 항목들 → L6 노드 (부모 = 첫 번째 information 노드)
5. Connection 생성: AI가 식별한 cross-link를 DB에 저장
6. 캐시 무효화: React Query 캐시 갱신 → UI 즉시 반영
```

#### 3.1.4 노드 표시 형식

| 레벨 | 표시 형식 | 예시 |
|------|-----------|------|
| L2 (Article) | `📄 {제목}` | 📄 양자역학의 본질과 핵심 원리 |
| L3 (지혜) | `💡 지혜 : {제목}` | 💡 지혜 : 관측의 영향 |
| L4 (지식) | `📖 지식 : {제목}` | 📖 지식 : 플랑크의 에너지 양자화 |
| L5 (정보) | `ℹ️ 정보 : {제목}` | ℹ️ 정보 : 양자역학이란 무엇인가 |
| L6 (데이터) | `📊 데이터 : {제목}` | 📊 데이터 : 플랑크 상수 |

### 3.2 방사형 마인드맵 (Radial Mind Map)

#### 3.2.1 레이아웃 알고리즘

- **배치 방식:** 중심(Cogito)에서 동심원(ring) 형태로 확장하는 방사형 트리
- **기본 반경:** 240px (첫 번째 링), 이후 링마다 220px씩 증가
- **최소 호 간격:** 100px (노드 간 최소 거리 보장)
- **가중치 시스템:** 자식 노드 수에 비례하여 각도 배분 (최소 가중치 3)
- **동적 반경 조절:** 노드가 겹치지 않도록 링 반경 자동 확대

```
계산 공식:
- 각 노드의 가중치 = max(하위 리프 노드 수, 3)
- 각도 비율 = 노드 가중치 / 전체 형제 가중치 합
- 위치: x = centerX + r × cos(θ), y = centerY + r × sin(θ)
- 반경: r = max(기본 반경, 최소호간격 / 최소각도차)
```

#### 3.2.2 SVG 렌더링 요소

| 요소 | 설명 |
|------|------|
| **중심 노드** | "Cogito / 나" 라벨, 바이올렛 보더, 카드 스타일 |
| **동심원 가이드** | 점선 원형 링, 낮은 투명도로 깊이감 표현 |
| **부모-자식 엣지** | 3차 베지에 곡선 (Cubic Bezier), 레벨 색상 적용 |
| **Cross-link 엣지** | 2차 베지에 곡선, 점선 스타일, 낮은 투명도 |
| **카테고리 노드 (L1)** | 보더+카드 스타일 foreignObject 박스 |
| **Article 노드 (L2)** | 바이올렛 보더-2, 그라데이션 배경, 📄 FileText 아이콘 |
| **DIKW 노드 (L3-6)** | 작은 보더/카드 박스, `{계층명} : {제목}` 형식 |
| **중심 글로우** | radialGradient 효과, 바이올렛 은은한 발광 |

#### 3.2.3 인터랙션

| 인터랙션 | 동작 | 입력 |
|----------|------|------|
| **노드 클릭** | 노드 상세 패널 열기 | 좌클릭 |
| **노드 더블클릭** | 해당 노드로 줌인 (포커스 이동) | 더블클릭 |
| **노드 드래그** | 개별 노드 위치 이동 (관성 물리) | 좌클릭 드래그 |
| **Article 클릭** | 원문 전체 보기 팝업 | L2 노드 클릭 |
| **팬(Pan)** | 뷰 전체 이동 | Space+드래그, Ctrl+드래그, 휠클릭+드래그 |
| **줌(Zoom)** | 뷰 확대/축소 | Ctrl+스크롤, 줌 버튼 |
| **뷰 리셋** | 전체 노드가 보이도록 자동 맞춤 | Maximize 버튼 |

#### 3.2.4 드래그 물리 엔진

노드 드래그에 **관성(Momentum)** 을 적용하여 자연스러운 UX를 구현한다.

```
물리 파라미터:
- 마찰 계수 (FRICTION): 3.5
- 속도 감쇠: v(t) = v₀ × e^(-FRICTION × Δt)   (지수적 감쇠)
- 정지 임계치: speed < 0.5px/s
- 드래그 감지 임계치: 3px (이하 이동은 클릭으로 처리)
- 렌더링: requestAnimationFrame 기반 60fps 애니메이션 루프
- 좌표 변환: screenToSvg() — SVG의 getScreenCTM().inverse() 활용
```

### 3.3 뷰 모드

#### 3.3.1 마인드맵 뷰 (기본)

- **전체 화면(Fullscreen)** SVG 방사형 마인드맵
- 플로팅 UI 컨트롤:
  - **좌상단:** SchemaMind 로고 + 앱 타이틀
  - **상단 중앙:** 브레드크럼 네비게이션 (줌인 시)
  - **우상단:** 뷰 전환 버튼 / AI 분석 버튼 / 다크모드 토글
  - **우측 하단 (맵 내):** 줌인/줌아웃/맞춤 버튼
  - **좌하단:** 노드 수 / 연결 수 통계

#### 3.3.2 그리드 뷰

- 카드 기반 격자 레이아웃 (1~3열 반응형)
- 현재 깊이의 자식 노드를 카드로 표시
- 브레드크럼 네비게이션으로 계층 탐색
- 수동 노드 추가 버튼 지원

### 3.4 노드 상세 패널

노드 클릭 시 우측에서 슬라이드인하는 상세 정보 패널:

- **헤더:** 계층 레이블 (예: `지혜 · L3`), 레벨 컬러 인디케이터
- **제목:** 노드 제목 (대형 볼드 텍스트)
- **설명:** 간략한 요약 텍스트
- **Content:** 상세 내용 (코드 블록 스타일 카드)
- **생성일:** 타임스탬프
- **Connections:** 해당 노드와 연결된 다른 노드 목록 (레벨 컬러 인디케이터 포함)
- **삭제:** 노드 삭제 버튼 (하위 노드 재귀 삭제)

### 3.5 연결(Connection) 관리 패널

- 전체 연결 목록 조회
- 수동으로 두 노드 간 연결 생성
- 연결 사유(description) 입력
- 연결 삭제

### 3.6 원문 전체 보기 팝업

Article(L2) 노드 클릭 시 표시되는 풀텍스트 오버레이:

- 반투명 배경 + 백드롭 블러
- Spring 애니메이션 모달
- 바이올렛 그라데이션 헤더
- 스크롤 가능한 본문 영역
- 닫기 버튼 및 배경 클릭 닫기

### 3.7 다크 모드

- **토글 버튼:** 태양(Sun) / 달(Moon) 아이콘 전환
- **저장:** localStorage에 테마 설정 저장
- **시스템 감지:** 초기 로딩 시 OS 다크모드 설정 자동 감지
- **적용 방식:** `document.documentElement` 에 `dark` 클래스 토글
- **CSS 변수:** Tailwind CSS의 `:root` / `.dark` 클래스 기반 색상 시스템

## 6. 사용자 시나리오 (User Flow)

### 6.1 텍스트 분석 플로우

```
1. 사용자가 마인드맵 화면에서 [AI 분석] 버튼 클릭
2. 분석 다이얼로그 열림
   - 9개 카테고리 안내 표시
   - DIKW 위계 설명 표시
   - 텍스트 입력 영역 (최대 50,000자, 실시간 글자 수 표시)
3. 사용자가 텍스트 붙여넣기 후 [분석 시작] 클릭
4. 로딩 상태 (스피너 + "AI 분석 중..." 표시)
5. AI 분석 완료:
   - 다이얼로그 자동 닫힘
   - 토스트 알림: "[카테고리] 제목 — N개 노드 생성"
   - 마인드맵에 새 노드들이 즉시 나타남
6. 사용자가 새로 생성된 노드를 탐색
```

### 6.2 마인드맵 탐색 플로우

```
1. 초기 화면: 중심 "Cogito"와 9개 카테고리 노드 표시
2. 카테고리 노드 더블클릭 → 해당 카테고리로 줌인
   - 브레드크럼 표시: Cogito > 자연과학
   - 해당 카테고리 하위 Article 노드들 표시
3. Article 노드 클릭:
   - (a) 노드 박스 자체 클릭 → 원문 전체 보기 팝업
   - (b) 노드 선택 → 우측 상세 패널 열림
4. DIKW 노드 더블클릭 → 해당 하위 계층으로 줌인
5. 브레드크럼으로 상위 계층 복귀 가능
6. [Maximize] 버튼으로 전체 뷰 리셋
```

### 6.3 노드 드래그 플로우

```
1. 노드 위에서 마우스 좌클릭 유지
2. 3px 이상 이동 시 드래그 모드 활성화
   - 커서 → grabbing
   - 노드 scale(1.05) + ring 효과
3. 마우스 이동에 따라 노드 위치 실시간 추적
   - SVG 좌표계로 변환 (screenToSvg)
   - 마지막 포인터 위치로부터 속도 계산
4. 마우스 릴리스:
   - 마지막 계산된 속도로 모멘텀 적용
   - 지수적 감쇠 (FRICTION=3.5)
   - speed < 0.5px/s 이하에서 정지
5. 드래그 중 부모-자식 엣지가 실시간으로 따라감
```
